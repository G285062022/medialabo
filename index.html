「Webコンテンツ作成」の成果

本テーマの学習目的

この学習テーマでは、HTML、CSS、JavaScriptを用いて、 ウェブページの構造を定義し、 文字のスタイルや色を変更し、画像などの要素を追加します。 また、DOMを利用してコンテンツの内容を読み取りや変更を行い、 ボタンやテキスト入力などの要素を追加して、ウェブページを構築していきます。 これにより、ウェブ開発の基本的な概念やスキルを身につけることを目指します。

プロジェクト課題について

プロジェクト課題では、世界の天気を表示するウェブページを作成しました。 ユーザーは都市名のIDを入力して検索することで、都市名、最高気温、最低気温、湿度が表示されます。 さらに、検索ボタンを押すと背景色がランダムにパステルカラーに変わる機能を実装しました。 また、都市名、最高気温、最低気温に合わせて絵文字が自動的に変わるようにしました。 ユーザーが都市のIDを知らなくても利便性を高めるため、天気を調べることが可能な都市のIDをページに記載しました。 これにより、ユーザーは気軽に興味のある都市の天気情報を調べることができます。 開発過程では、API連携と非同期処理の理解が重要でした。 また、背景色や絵文字の自動切り替えの実装にも工夫が必要でした。 デザイン面では、ページの見た目や使いやすさを考慮し、シンプルで直感的なデザインを採用しました。 今後はユーザーフィードバックを収集し、エラーハンドリングやデザインの改善を行い、 より使いやすく魅力的なウェブページにしていきたいと考えています。 また、新たな機能の追加や地域の天気情報の表示など、さらなる拡張も検討しています。

    • 数当てゲーム

    数当てゲームを作成してとても楽しい経験でした。 JavaScriptを使ってユーザーとの対話的な体験を実現できることに感動しました。 DOM操作やイベントリスナーの設定、条件分岐などのプログラムの基本的な概念を学ぶことができ、 特に`addEventListener`を使ってボタンがクリックされた時の処理を設定する方法に興味を持ちました。 乱数を生成して毎回異なる問題を提示することで、プレイヤーが予想を入力し、結果が表示されるところがゲームの魅力でした。 DOM操作を通じてHTML要素の内容を変更し、プレイヤーにフィードバックを提供することで、ゲームの進行状況をわかりやすく伝えることができました。 ただし、今回作成したゲームは簡単なものであり、より多様な機能を追加することで、楽しさや興味を引き出す可能性を感じました。例えば、難易度の設定や制限時間の導入、得点システムの実装などが挑戦的な課題となるでしょう。 数当てゲームの制作を通じて、プログラミングの基本的なスキルの向上と創造性を発揮する楽しさを実感しました。これからもさまざまなアプリケーションを作成し、より多くのことを学んでいきたいと思います。

    今回は、 p{ color: green; }で文字を緑色にしたり、 p.info{ font-size: small; Background-color: #dfdfdf; }を使って文字を通常よりも小さくしたり、背景を灰色にしたりなど Cssについて学びました。他にも画像を右側に寄せたり、画像の幅を広くしたり、画像の上下左右の余白部分を設定する方法を学びました。

    第3回 JavaScript の基本

    今回の学習ではJavaScript の基本について学びました。console.log()は文字列を出力するための機能です。 JavaScriptプログラムはscript要素で指定し、defer属性を使うことで実行を先送りすることができます。 defer属性を指定すると、JavaScriptプログラムの実行はHTMLとCSSの読み込み後に行われます。 しかし、deferを忘れると読み込み順序が変わり、正しく動作しなくなることもあります。 また、script要素の終了タグは必ずつける必要があります。 onclick属性を使うことでプログラムを実行することも可能。 JavaScriptは長い歴史を持つ言語であり、WWWから情報を入手する際には注意が必要だと思います。

    • 練習問題3-6 の practice-js/obj-practice.html

    第4回 DOM と JavaScript

    第4回では、DOMの操作に使用するいくつかのメソッドや機能について学びました。 document.querySelectorとdocument.querySelectorAllは要素の検索に使用され、 textContentプロパティは要素のテキストを参照します。 span要素は特別な意味を持たず、インライン要素として利用されます。 document.createElementで新しい要素を作成し、insertAdjacentElementを使って要素を追加できます。 setAttributeを使うと要素の属性を設定できます。 また、removeメソッドを使用することで要素を削除できます。 これらの機能を使ってJavaScriptでDOMを操作することができます。

    • 練習問題4-2~5 の practice-dom/dom-exercise.html

    第5回 イベント処理

    イベントハンドラを使って、ユーザーの操作や要素の状態変化に応じた処理を実行する方法を学びました。 ボタンのクリックや入力フィールドの変化、ラジオボタンやチェックボックスの選択など、 特定のイベントが発生したときに関数が実行されます。 「event-intro.js」では、ボタンクリックで見出しのテキストの色を変える処理を、 「input-demo.js」では、ボタンクリックで入力フィールドの値をコンソールに表示する処理を、 「radio-checkbox-demo.html」では、 ラジオボタンやチェックボックスの選択状態をコンソールに表示する処理を定義しています。 これにより、ユーザーの操作に合わせて必要な処理が実行されます。

    工夫点、苦心点、強調点、感想

    HTML、CSS、JavaScriptを使用してWebページを作成する経験は初めてだったため、いくつかの苦労点や面白い発見がありました。 特に、プロジェクト課題でのDOM操作と数当てゲームの制作に苦労しました。 私は、プロジェクト課題で、世界の天気を表示するウェブページを作成しました。最初はAPI連携と非同期処理に戸惑いましたが、 理解を深めることで天気情報を表示するものを作成することができました。 数当てゲームでは、JavaScriptを使ってユーザーとの対話的な体験を実現しました。 DOM操作や条件分岐などのプログラムの基本的な概念を学びました。 特に`addEventListener`を使ってボタンがクリックされた時の処理を設定する方法に興味を持ちました。 しかし、ゲームの難易度や機能が限定的だったため、より多様な機能を追加することで楽しさや興味を引き出せる可能性を感じました。 今後はゲームの改善や新機能の追加に挑戦していきたいと思っています。 これらの作業において工夫した点は、全体的な色やデザインを統一し、見やすく使いやすいWebページを目指しました。 特にプロジェクト課題では背景色のランダム変化や絵文字の表示など、ユーザーにとって興味深く楽しい要素を盛り込むことを心掛けました。 一方で苦労した点は、特にDOM操作です。 要素の検索やテキストの変更、新しい要素の追加など、正確なコードの記述や要素の配置に苦労しました。 また、数当てゲームでも表示の設定や条件分岐に手間取りましたが、説明を聞くことで理解を進めることができました。 全体的な感想としては、HTML、CSS、JavaScriptを使ってWebページを作成することは初めてでしたが、 多くの知識とスキルを身につけることができ、プログラミングの面白さと奥深さを実感しました。最初は難しく感じた部分もありましたが、 課題に取り組むうちに徐々に理解が深まっていく過程が楽しかったです。自分の考えを実現するためにコードを書く過程は創造的でやりがいがあります。 今後はさらに知識を深め、より複雑で魅力的なWebページやアプリケーションを作成していきたいと考えています。 また、新しい技術や開発手法にも積極的に取り組んで、より高度なプロジェクトに挑戦していきたいと思いました。 なぜ私の”index.html”と”GitHub”がうまく連携されないのかを授業が終わってしまったいまでも謎なところが残念でした。 これからも学習をして解決したいと思いました。
